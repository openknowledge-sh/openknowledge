Open Knowledge CLI

Local LLM wiki tooling for agents and humans

Create Git-native project knowledge bases in plain Markdown. Set them up with local agents, validate the structure, search exact source-grounded context, and publish portable static docs.

Let an agent create the wiki

Paste the prompt into Codex, Claude, Cursor, Cowork, or another coding agent in the workspace where the wiki should live.

Set up an Open Knowledge LLM wiki for this workspace. First check whether the openknowledge CLI is available with command -v openknowledge and openknowledge --help. If it is missing, install it with curl -fsSL https://openknowledge.sh/install | bash. Then run openknowledge setup, use openknowledge rules --list to see the available maintenance rules, inspect this workspace and any relevant memories, ask only the setup questions still needed, choose the maintenance rules this wiki should follow, such as project, docs, decisions, changelog, research, bugs, schemas, summary, or agents, create and customize the wiki for this workspace, run openknowledge validate, and show me how to inspect and navigate it with openknowledge list, openknowledge search, openknowledge get, and openknowledge view.

Prefer shell setup? Run curl -fsSL https://openknowledge.sh/install | bash, then openknowledge setup. The README has the full command reference.

At a glance

One Markdown wiki, several operational views

The same OKF bundle stays readable in Git, useful to agents, and inspectable through deterministic CLI commands.

setup

Agent setup

setup, from, and rules print prompts that let local agents create and maintain useful project memory.

files

Plain Markdown

Knowledge stays in Git-friendly files that humans can read and agents can patch without a database or hosted service.

use

Retrieval

search, get, list, and view make the wiki usable as local context instead of passive docs.

ship

Portable publishing

HTML exports include llms.txt, openknowledge.json, and a bundle archive so published wikis can be connected again.

check

Deterministic checks

validate, ast, JSON, graph, and experimental agent job commands provide structured views automation can trust.

okf

OKF v0.1

Open Knowledge implements the Open Knowledge Format v0.1 Markdown and YAML-frontmatter spec.

Command map

Use the CLI by layer

Start with prompts, inspect the bundle, connect stable names, then publish or export the same source files.

Layer Commands Use them for
Agent setup setup, rules, review rules Print prompts and maintenance instructions for agents that create or maintain a wiki.
Source-to-wiki from Turn a source URL or path into a local OKF Markdown wiki through an agent task prompt.
Format hygiene new, spec, validate, list, ast Create bundles, inspect structure, parse Markdown, and enforce portable OKF rules.
Registry connect, disconnect, registry, to tar Give local, published, archive, or Git knowledge bases stable names and portable archives.
Use and navigation get, search, list, view Read exact Markdown files, inspect bundle trees, search chunks, and browse locally.
Views and publishing to json, to graph, to html, to html --plain Export normalized models, graphs, static viewers, and plain semantic HTML.
Experimental automation agents Validate, dry-run, and execute scheduled local agent jobs from Markdown specs.
