Why Open Knowledge

Open Knowledge is a small tooling stack around Markdown knowledge bases. It is useful when you want local project memory that stays inspectable with normal shell tools.

  • Plain-file knowledge bases that agents and humans can both edit.
  • Source-to-wiki prompts for turning repositories, local folders, or websites into OKF Markdown bundles.
  • Deterministic validation, listing, search, AST, JSON, graph, tar, and HTML views of the same bundle.
  • Local registry aliases so agents can address knowledge bases by stable names.
  • Static publishing with connect manifests and portable bundle archives.

Use the CLI by layer

Layer Commands Use them for
Agent setup setup, rules, review rules Print prompts and maintenance instructions for agents that create or maintain a wiki.
Source-to-wiki from Turn a source URL or path into a local OKF Markdown wiki through an agent task prompt.
Format hygiene new, spec, validate, list, ast Create bundles, inspect structure, parse Markdown, and enforce portable OKF rules.
Registry connect, disconnect, registry, to tar Give local, published, archive, or Git knowledge bases stable names and portable archives.

Experimental automation agents Validate, dry-run, and execute scheduled local agent jobs from Markdown specs.

Common Workflows

Five short loops cover the README path: create the bundle, generate one from a source, connect it, inspect it, then publish or export the same files.

Create a bundle

Start a local OKF Markdown knowledge base.

openknowledge new ./project-memory
openknowledge new --no-agents --no-setup ./source-wiki

Create from source

Print a source-to-wiki prompt, then paste it into your agent.

openknowledge from https://github.com/openknowledge-sh/openknowledge --out Wiki --type understanding
openknowledge from https://openknowledge.sh/wiki/ --out Wiki --type understanding --depth 2

Connect and navigate

Give the bundle a stable name and browse it locally.

openknowledge connect ./project-memory --as personal
openknowledge search personal "validation workflow"
openknowledge view personal

Validate and inspect

Check structure and inspect the parsed Markdown model.

openknowledge validate ./project-memory
openknowledge list --depth 2 ./project-memory
openknowledge ast ./project-memory

Publish or export

Ship static docs or export normalized machine-readable views.

openknowledge to html --out ./project-site ./project-memory
openknowledge to json ./project-memory
openknowledge to tar --out ./project-memory.tar.gz ./project-memory
