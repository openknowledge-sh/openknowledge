Create a local LLM wiki for you and your agents

Open Knowledge CLI creates a Git-native local LLM wiki for project context that humans and agents can read, maintain, and publish.

Implements the Open Knowledge Format v0.1 specification, a Markdown and YAML-frontmatter standard that stays readable for humans, shell tools, and agents.

Set up an Open Knowledge LLM wiki for this workspace. First check whether the openknowledge CLI is available with command -v openknowledge and openknowledge --help. If it is missing, install it with curl -fsSL https://openknowledge.sh/install | bash. Then run openknowledge setup, use openknowledge rules --list to see the available maintenance rules, inspect this workspace and any relevant memories, ask only the setup questions still needed, choose the maintenance rules this wiki should follow, such as project, docs, decisions, changelog, research, bugs, schemas, summary, or agents, create and customize the wiki for this workspace, run openknowledge validate, and show me how to inspect and navigate it with openknowledge list, openknowledge search, openknowledge get, and openknowledge view.

Need manual installation, or advanced setup options? See the GitHub docs.

What an Open Knowledge LLM wiki gives you

A local Markdown LLM wiki that lives with your project and gives agents a clear place to read, update, and preserve durable context.

  • Turn a project or research folder into an agent-readable LLM wiki stored as normal Markdown in Git.
  • Use it as a Karpathy-style LLM wiki for project memory without giving up plain files, Git history, or portable parsing.
  • Guided setup through an agent interview, so the wiki starts with the right purpose, structure, and maintenance rules like docs, changelog, decisions, or research.
  • Use and search commands help agents load entrypoints, find source-grounded context, and navigate related wiki areas as project knowledge grows.
  • A maintenance loop that keeps decisions, workflows, and project memory current as work changes.
  • A local Markdown viewer for browsing, searching, and inspecting the wiki.
  • AST, JSON, source graph, and search graph outputs expose different views of the same OKF bundle.
  • Consistency against the Open Knowledge Format v0.1 specification.
