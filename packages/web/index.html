Create a local LLM wiki with your coding agent

Open Knowledge helps you create and maintain local LLM wikis that are both human-readable and agent-readable. It keeps your knowledge base portable and consistent using the Open Knowledge Format v0.1 spec.

Paste one prompt into Codex, Cowork, Cursor, Claude, or another coding agent. The agent installs the CLI, asks where the wiki should live, creates the scaffold, customizes workflows, and validates the result.

Set up an Open Knowledge agentic wiki for this workspace.

First make sure the Open Knowledge CLI is installed. If it is not installed,
install it with:

curl -fsSL https://openknowledge.sh/install | bash

Then run:

openknowledge setup

Read the setup guide printed by that command and follow it. Ask me the setup
questions, create the knowledge base with openknowledge new, customize it for
this workspace, create useful workflows/skills/automation specs, run
openknowledge validate, and show me how to inspect it with openknowledge list
and openknowledge open.

What Open Knowledge gives you

A wiki can live inside a project repo or stand alone as a private knowledge base. It stays readable in Git and easy for both humans and coding agents to navigate.

  • Portable Markdown files with predictable names, frontmatter, indexes, and logs.
  • Agent-readable scaffolding with AGENTS.md, workflows/, skills/, automations/, and a pinned SPEC.md.
  • A local Markdown viewer plus validation against embedded OKF v0.1 rules.

Codex CLI shortcut

Interactive Codex needs stdin to remain a terminal, so pass the generated setup prompt as an argument: 

codex "$(openknowledge setup)"

Manual setup

If you want the lower-level path, install the CLI and create a generic scaffold directly: 

curl -fsSL https://openknowledge.sh/install | bash

npm install -g @openknowledge-sh/openknowledge

openknowledge new ./project-memory
openknowledge open ./project-memory
openknowledge list ./project-memory
openknowledge validate ./project-memory

Commands

  • setup prints an agent prompt for creating and customizing a knowledge base.
  • new scaffolds a local Open Knowledge bundle.
  • open starts a local Markdown viewer for the bundle.
  • list prints the bundle tree with inline validation issues.
  • validate checks the bundle against the latest or selected spec.
  • spec prints the embedded Open Knowledge Format spec.
